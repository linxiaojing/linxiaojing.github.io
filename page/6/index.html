各种搭建博客平台程序的对比

面对现在这么多的独立搭建博客程序，如何进行选择呢？为了减少折腾时间，总结了一下各程序的优劣。

1、Wordpress，搭建平台PHP+MYSQL；

  • 优点：搭建简单，功能强大，插件及主题模板资源非常多，备份简单，支持转到多种其它博客程序；
  • 缺点：程序越变越臃肿，且需在线编辑文档（在线编辑的一个问题就是需排版，而且写作时不小心刷新就会丢失文档）。wordpress已经看起来越来越不像是一个博客了，更像是一个CMS。

2、Jekyll，使用Ruby语言，搭建平台Ruby+devkit+gem，如果使用博客的代码高亮用到了Python的Pygments模块，则需再安装Python；

  • 优点：主题模板较多，支持markdown、liquid以及原始的html、css， 可直接在github上搭建， 继承github上的各种好处（版本控制，免费，无流量限制）；
  • 缺点：安装相对复杂（使用jekyllbootstrap框架安装则相对简单，不过不支持windows下搭建），仅仅适合小型网站，不适合大中型网站；
  • 即是优点也是缺点是它生成的是静态页面无法实现动态加载，不需要额外的数据库（不方便数据搜索）。

3、Octopress，基于Jekyll，同样使用Ruby实现，搭建平台Ruby+devkit+git，如果使用博客的代码高亮用到了Python的Pygments模块。其优缺点与Jekyll类似。

推荐一些简洁的Hexo主题模板

博客和其它网站不同，它不是以网站显示效果来吸引眼球，而是以其内容来吸引眼球，所以博客就是应该回归于写作的初衷。所以建个博客，最好具备“写作简单”、“发表简单”、“模板简单”这3个条件。

  • “写作简单”就是指可以通过Markdown直接本地PC、手机、iPad上流畅地写作，以便使用Evernote或是Dropbox等进行随时随地的同步；
  • “发表简单”指的是可以不用再排版或是增加任何格式就可直接发表，现在第三方提供博客平台的服务，很多都是使用Evernote或是Dropbox的API接入服务，使得通过Evernote或是Dropbox就可直接发表博客文章，如Postach.io+Evernote，pancake.io+Dropbox，FarBox.com+Dropbox，除此外FarBox还提供绑定微信服务（随时随地，几句话，或几张图，就可汇集成一天的日记）；
  • “模板简单”，就是网站不用太过于花俏，应该把重点用于内容上。除了内容外，主要加上自己的介绍和社交网站的一些ID链接即可。现在基本流行的简单模板，如blog.farbox.com样式。

推荐几个Hexo的主题模板：NexT {下载地址}、carbon{下载地址}、Kael{下载地址}、scribble{下载地址}、Yilia {下载地址}。

关于域名自动续费这件事

国外域名注册网站一般都会在域名注册后自动帮你把域名设置为自动续费，我就因此吃过很多次亏，明明是自己不想续费的域名，忘记更改设置，因而产生了不必要的扣费。今天三八妇女节的早上，这种悲具再次发生了。

我于去年在101domain.com上注册了13个.club域名，这些域名基本很多都是学费米（即注册后过段时间发现基本没什么价值的域名），而昨天收到离域名过期日还有60天的邮件短信通知，但是我并没有具体去查看邮件的内容。今天一早这13个域名全部都自动续费了，每个是13.37美元，共计扣了我信用卡153.86美元。而根据它们网站上的规定，凡是自动续费成功的，是没办法再申请退款的。所以也只能认了。

记得前年也吃过这样的亏，当时是在Godaddy上的auctions member会员自动续费4.99美元忘记取消了。今年亏更大，真是屡教不改呀。再次提醒自己，一定要检查好这些网站的自动续费设置，今后一定要避免再发生类拟这样的事情了。

升级Hexo到3.0版本

昨天hexo发布了3.0版本，今天根据其升级说明Migrating-from-2.x-to-3.0进行了升级，升级时需注意因很多模块都分割独立出来了，所以安装时需再安装server、deployers等模块才能用，插件也需进行重新升级，等你全部照文档安装升级后会在deploy时会出现：

ERROR Deployer not found: github

解决方法：更改_config.yml文件中

deploy:
       type: github

为

deploy:
       type: git

end

知识的管理和电脑软件整理

###我的知识管理方面：
以前关于知识的管理，我并没有做过太多的研究，很简单地就是使用印象笔记来分类记录知识要点，而后在简书里看到CnFeat写的一篇“个人知识管理的方法”再得到两点重要的应用：其一是Chrome浏览器里安装「印象笔记·悦读」插件，非常方便剪辑自己看到优秀文章直接摘入印象笔记（所以根本不需要如pocket类的稍后阅读软件；其二是通过关注“我的印象笔记”微信公众号，即可收录微信看到的优秀文章，不过遗憾的是它不支持印象笔记intelnational版，所以我只能再注册一个国内的账号。

关于云盘和移动APP：

1、云盘：因Dropbox在国内需翻墙，所以我到现在基本放弃使用了；目前我使用的是两个，一是百度云盘，用于同步我的工作文件和私人文件，一是坚果云，和我老婆共用，同步宝宝的照片等;

2、移动APP：主要指阅读工具（包括新闻和推特类），目前频率使用最高的是Zaker、简书、Falcon pro、Google+、微信，建议不要安装得太多，这样容易走马观花，无法集中注意力;
